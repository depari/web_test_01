         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">First Public Working Draft: WCAG 2.2</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-27T13:29:00Z"> 27 February 2020 </span> | <a title="Archive: First Public Working Draft: WCAG 2.2" href="https://www.w3.org/blog/news/archives/8334">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The Accessibility Guidelines Working Group (<a href="https://www.w3.org/WAI/GL/">AG WG</a>) has published a First Public Working Draft of <a href="https://www.w3.org/TR/2020/WD-WCAG22-20200227/">Web Content Accessibility Guidelines (WCAG) 2.2</a>. WCAG provides recommendations for making web content more accessible to people with disabilities. It addresses accessibility of web content on desktops, laptops, tablets, and mobile devices. Following these guidelines also makes your web content more usable to <a href="https://www.w3.org/WAI/perspective-videos/">all users in a variety of situations</a>. Please see the blog post for information on what&#x2019;s new in this draft and upcoming work: <a href="https://www.w3.org/blog/2020/02/wcag-22-in-development/">Web Content Accessibility Guidelines (WCAG) 2.2 in Development</a>.</p>
</div>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">W3C awards Website Redesign Project to Studio 24</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-25T13:02:32Z"> 25 February 2020 </span> | <a title="Archive: W3C awards Website Redesign Project to Studio 24" href="https://www.w3.org/blog/news/archives/8326">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p><a class="imageLink" href="/2020/02/pressrelease-studio-24-w3c-website-redesign.html.en"><img src="/Icons/WWW/w3c_home_nb.png" alt="W3C" height="48" width="72" /><img src="/2020/02/studio-24-logo.png" alt="Studio 24" height="48" width="48" /></a> The <a href="/2019/11/website-redesign-rfp.html">W3C Website redesign Request For Proposals</a> has concluded with W3C awarding the project to <a href="https://www.studio24.net/">Studio 24</a>, a small, independent digital agency in the heart of Cambridge, UK, founded in 1999. The project covers a subset of the public-facing pages of our Website and will span the next 10 months. Please, read more in our <a href="/2020/02/pressrelease-studio-24-w3c-website-redesign.html.en">joint press release</a>.</p>
<p>We are aiming for <a href="/2019/11/website-redesign-rfp.html#phases">this phase</a> to scale well to cover the redesign expectations for the rest of the site. Future phases will include the Member and Team spaces, internal Work Groups homepages, specifications template, mailing lists archives, W3C Community Groups and Business Groups.</p>
<p>The current website was implemented <a href="/News/2009#entry-8628">ten years ago</a> and is no longer as effective in supporting W3C&#x2019;s mission and goals as it could be. We believe that by <strong>implementing current web best practices and technologies, revising the information architecture, creating a content strategy and revamping the visual design</strong>, we can provide our audiences with the best information in a more user-friendly fashion, motivate participation in the organization, and communicate the nature and impact of W3C more effectively.</p>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">First Public Working Draft: XR Accessibility User Requirements</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-13T07:43:39Z"> 13 February 2020 </span> | <a title="Archive: First Public Working Draft: XR Accessibility User Requirements" href="https://www.w3.org/blog/news/archives/8320">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>
The <a href="https://www.w3.org/WAI/APA/">Accessible Platform Architectures Working Group</a> has published a First Public Working Draft of <a href="https://www.w3.org/TR/2020/WD-xaur-20200213/">XR Accessibility User Requirements</a>. This document lists user needs and requirements for people with disabilities when using virtual reality or immersive environments, augmented or mixed reality and other related technologies (XR). This document is most explicitly not a collection of baseline requirements. It is also important to note that some of the requirements may be implemented at a system or platform level, and some may be authoring requirements.</p>
</div>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">First Public Working Drafts: Resize Observer; CSS Scroll Anchoring Module Level 1</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-11T09:12:32Z"> 11 February 2020 </span> | <a title="Archive: First Public Working Drafts: Resize Observer; CSS Scroll Anchoring Module Level 1" href="https://www.w3.org/blog/news/archives/8311">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>
The <a href="https://www.w3.org/Style/CSS/">CSS Working Group</a> has published two First Public Working Drafts today:
</p>
<ul class="show_items">
<li><a href="https://www.w3.org/TR/2020/WD-resize-observer-1-20200211/">Resize Observer</a>: This specification describes an API for observing changes to Element&#x2019;s size.</li>
<li><a href="https://www.w3.org/TR/2020/WD-css-scroll-anchoring-1-20200211/">CSS Scroll Anchoring Module Level 1</a>: Changes in DOM elements above the visible region of a <a href="https://drafts.csswg.org/cssom-view-1/#scrolling-box">scrolling box</a> can result in the page moving while the user is in the middle of consuming the content. This spec proposes a mechanism to mitigate this jarring user experience by keeping track of the position of an anchor node and adjusting the scroll offset accordingly.</li>
</ul>
<p>
<a href="https://www.w3.org/TR/CSS/">CSS</a> is a language for describing the rendering of structured documents (such as HTML and XML) on screen, on paper, etc.</p>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Trace Context is a W3C Recommendation</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-06T14:07:20Z"> 6 February 2020 </span> | <a title="Archive: Trace Context is a W3C Recommendation" href="https://www.w3.org/blog/news/archives/8298">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>
The <a href="https://www.w3.org/2018/distributed-tracing/">Distributed Tracing Working Group</a> has published <a href="https://www.w3.org/TR/2020/REC-trace-context-1-20200206/">Trace Context</a> as a W3C Recommendation. This specification defines standard HTTP headers and a value format to propagate context information that enables distributed tracing scenarios. The specification standardizes how context information is sent and modified between services. Context information uniquely identifies individual requests in a distributed system and also defines a means to add and propagate provider-specific context information.</p>
</div>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">W3C Workshop Report: Inclusive Design for Immersive Web Standards</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-06T13:22:24Z"> 6 February 2020 </span> | <a title="Archive: W3C Workshop Report: Inclusive Design for Immersive Web Standards" href="https://www.w3.org/blog/news/archives/8302">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p><a class="imageLink" href="https://www.w3.org/2019/08/inclusive-xr-workshop/report.html"><img src="https://www.w3.org/comm/assets/graphics/incxr.svg" alt="" width="180" /></a>
W3C is pleased to announce a <a href="https://www.w3.org/2019/08/inclusive-xr-workshop/report.html">report</a> from the <a href="https://www.w3.org/2019/08/inclusive-xr-workshop/">W3C Workshop on Inclusive Design for Immersive Web Standards</a>, held on 5-6 November 2019 in Seattle, WA, USA. 
</p>
<p>
This report contains a brief summary and collects highlights from the
individual sessions, with links to the presentation slides. Workshop participants learned from existing approaches that have been
taken in making XR experiences (on and off the Web) accessible before
looking at what lessons could be derived from these existing research
and experiments in the context of the Immersive Web architecture.
</p>
<p>
These lessons brought forward four aspects of accessible XR experiences:
visual interactions, motricity considerations, audio aspects and
assistive technologies adaptation.
</p>
<p>
The relevant follow-up work in W3C spans across at least 6
standardization Working Groups and 6 pre-standardization and incubation
Community Groups, and also intersects with at least 3 Khronos Working
Groups &#x2013; pointing toward the need for a strong coordination effort to
ensure systematic and consistent progress for the Web platform. We
propose to host this coordination in the Inclusive Design for the
Immersive Web Community Group via a <a href="https://github.com/w3c/inclusive-xr-roadmap/issues">dedicated github repository</a>.
</p>
<p>
We thank our host, Pluto VR, Maveron, We Make Reality, Virtual World
Society and Seattle Immersive Technology Association, our sponsors,
Google, Twitch and Samsung Internet, the <a href="https://www.w3.org/2019/08/inclusive-xr-workshop/#committee">Program Committee</a>, and all
<a href="https://www.w3.org/2019/08/inclusive-xr-workshop/participants.html">participants</a> for making this event possible.</p>
</div>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Data Catalog Vocabulary (DCAT) version 2 is a W3C  Recommendation</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-02-04T11:26:49Z"> 4 February 2020 </span> | <a title="Archive: Data Catalog Vocabulary (DCAT) version 2 is a W3C  Recommendation" href="https://www.w3.org/blog/news/archives/8287">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The <a href="https://www.w3.org/2017/dxwg/">W3C Dataset Exchange Working Group</a> has published <a href="https://www.w3.org/TR/2020/REC-vocab-dcat-2-20200204/">Data Catalog Vocabulary (DCAT) &#x2013; Version 2</a> as a W3C Recommendation. DCAT defines an RDF vocabulary designed to facilitate interoperability between data catalogs published on the Web. The specification defines the schema and provides examples for its use. Version 2 extends the original DCAT standard in line with community practice while supporting diverse approaches to data description and dataset exchange.</p>
</div>

      <div class="headline">
         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Postponed: W3C Workshop on Web &amp; Machine Learning</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2020-01-14T07:58:48Z"> 14 January 2020 </span> | <a title="Archive: Postponed: W3C Workshop on Web &amp; Machine Learning" href="https://www.w3.org/blog/news/archives/8238">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>
<a class="imageLink" href="https://www.w3.org/2020/01/machine-learning-workshop/"><img src="https://www.w3.org/blog/news/files/2020/01/machine-learning-workshop.png" alt="W3C Web &amp; Machine Learning Workshop page banner image" width="150" /></a> <strong>2020-02-13 Update:</strong> Given the current challenges and restrictions around international travel, the Program Committee has made the decision to postpone the workshop to a later date.</p>
<p>W3C announced today a <a href="https://www.w3.org/2020/01/machine-learning-workshop/">Workshop on Web &amp; Machine Learning</a>, 24-25 March 2020, in Berlin, Germany. The event is hosted by Microsoft.
</p>
<p>
The primary goal of the workshop is to bring together providers of Machine Learning tools and frameworks with Web platform practitioners to enrich the Open Web Platform with better foundations for machine learning.
</p>
<p>The secondary goals of the workshop are as follows:</p>
<ul class="show_items">
<li>Understand how machine learning fits into the Web technology stack,</li>
<li>Understand how browser-based machine learning fits into the machine learning ecosystem,</li>
<li>Explore the impact of machine learning technologies on Web browsers and Web applications,</li>
<li>Evaluate the opportunities for standardization around machine learning APIs and formats.</li>
</ul>
<p>Expected topics of discussion include:</p>
<ul class="show_items">
<li>Dedicated machine learning APIs for browsers as explored by the <a href="https://www.w3.org/community/webmachinelearning/">Machine Learning for the Web Community Group</a></li>
<li>Integration of browser-provided data sources in machine learning workflows</li>
<li>Layering and interoperability of machine learning APIs with other computing APIs (e.g. WebGPU, WebAssembly)</li>
<li>Domain-specific machine learning APIs</li>
<li>Interchange format for machine learning models on the Web</li>
<li>Client- and cloud-based machine learning interactions</li>
<li>On-device machine learning training in browsers</li>
<li>Risks and benefits of browser-based machine learning on privacy, security, accessibility</li>
<li>Using machine learning primitives to help improve accessibility of Web pages and applications</li>
<li>Machine learning frameworks for the Web</li>
<li>Machine learning hardware accelerators</li>
</ul>
<p>
Attendance is free for all invited participants and is open to the public, whether or not W3C members. For more information on the workshop, please see the <a href="https://www.w3.org/2020/01/machine-learning-workshop/">workshop details and submission instructions</a>.
</p>
<p>
<a href="https://www.w3.org/2002/09/wbs/1/machine-learning-workshop/">Registration</a> is available online due by <strong>21 February 2020</strong>.</p>
</div>
